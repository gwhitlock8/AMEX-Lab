Section 2.1: Build the Knowledge Gap Analyzer Agent #

Note In the following lab, we will talk through prompting many times. To learn more about prompting in ServiceNow, visit our community and our Product Manager's excellent guide on prompting.

Step 1: Create the Agent

  1. Open AI Agent Studio (All > AI Agent Studio > Overview)

  2. Locate the section titled "Recent use cases and AI agents activity". Select the AI Agents tab, then click "Add". Choose "Chat"

  3. A configuration page for "New AI agent" will open. In the Define the specialty area, click on 'Generate details.'

  4. Copy and paste the following into the dialog:

    I want an AI agent that analyzes knowledge base coverage gaps. This agent retrieves recent incidents from the incident table to understand what issues people are experiencing, then searches the knowledge base to see what documentation already exists. It compares the two datasets to identify incident categories and topics that have no corresponding knowledge articles. It presents a prioritized gap report showing: the topic, how many incidents it represents, and a recommended article title. Results are sorted by incident volume so teams know which articles to write first. The agent uses a professional, analytical tone and formats output as a clean numbered list. It ends with a summary of total incidents analyzed, gaps found, and estimated deflection impact.

  5. Click on "Generate." Scroll through the results and see how Now Assist has filled in the AI Agent role, list of steps, and Agent name.

    • You may see a "Possible duplications found" dialog pop up. Click "Ignore and continue."
Tip If you do not see the entry field for "AI Agent Role", you may be in the wrong section! Check the upper left corner of the screen and confirm that you are in the "New AI Agent" setup.

  1. Review the generated Instructions. Don't worry about perfecting them yet — we'll come back and refine these after we add the tools. For now, accept what Now Assist generated.

  2. Click "Save and continue"

Step 2: Add the Tools

Now we give the agent its capabilities. Both tools use the built-in Record operation — no custom development, no Flow Designer work required.

  1. Click Add tool and configure the first tool:

    • Select Record operation
    • Name: "Get Recent Incidents"
    • Description: "Retrieve recent incident records to analyze trending issues and identify categories lacking knowledge base coverage."
    • Table: Search for and select "Incident"
    • Select operation: "Look up records"
    • Conditions: click "+ Add condition set", then set Active — is — true
    • Max records returned: change from 5 to 50
    • Returned fields: select short_description, category, subcategory, priority
    • Order By: select Created (sys_created_on)
    • Sort Type: select Descending
    • Execution mode: Autonomous
    • Display output: Yes

    Under advanced settings:

    • Output transformation strategy: Concise
    • 'Generate messages' letting Now Assist fill in the responses, or enter:
      • Thinking: "Retrieving recent incidents..."
      • Completion: "Incident data retrieved."

  2. Click 'Add'

  3. Click Add tool again and configure the second tool:

    • Select Record operation
    • Name: "Search Knowledge Base"
    • Description: "Retrieve published knowledge articles to determine what documentation already exists. Use this to compare against incident topics and identify gaps."
    • Table: Search for and select "Knowledge" (kb_knowledge)
    • Select operation: "Look up records"
    • Conditions: click "+ Add condition set", then set Workflow state — is — Published
    • Max records returned: 50
    • Returned fields: select short_description, topic
    • Execution mode: Autonomous
    • Display output: Yes

    Under advanced settings:

    • Output transformation strategy: Concise
    • 'Generate messages' letting Now Assist fill in the responses, or enter:
      • Thinking: "Searching knowledge base..."
      • Completion: "Knowledge base search complete."

  4. Click Add

  5. Your tools list should now show two tools:

    • Get Recent Incidents (Record operation — Look up records) - Autonomous
    • Search Knowledge Base (Record operation — Look up records) - Autonomous

  6. Click Save and Continue

Step 3: Refine the Instructions

Now that our tools exist, we need to go back and make sure the agent's instructions tell it exactly how to use them. This is the most important step — the instructions are what make the agent smart.

  1. Navigate back to the agent's configuration. Click on the agent name or use the breadcrumb to return to the Define the specialty section.

  2. Replace the generated Instructions with the following. Notice how each step references the tools by the exact names we just created:

    Instructions 
    1. When asked to analyze knowledge gaps, use the "Get Recent Incidents" tool to retrieve recent incident data. This returns incident short descriptions and categories.
2. Use the "Search Knowledge Base" tool to retrieve published knowledge articles. Review the returned articles and compare them against the incident topics you identified.
3. Compare the incident topics against the knowledge articles returned. Identify incident categories or topics where NO relevant knowledge article exists.
4. Present your findings as a prioritized list. For each gap, include: the topic/category, how many incidents relate to it, and a recommended article title. Sort by incident count (highest first).
5. End with a brief summary: total incidents analyzed, number of gaps found, and estimated incident deflection if the top 3 articles were created.
Key Takeaway

This is how prompt engineering works with AI Agents. The tools give the agent capabilities, but the instructions tell it when and how to use them. Without instructions that reference tool names, the agent won't know it has tools to call. This refine-after-building pattern is how you'll iterate on agents in the real world too.

  1. Click Save

Step 5: Test the Agent

  • In the Task box, enter "Analyze our knowledge base for gaps based on recent incidents" and click Continue to Test Chat response.
  • The agent should now:
    • Call the "Get Recent Incidents" tool to pull recent incident data
    • Call the "Search Knowledge Base" tool to retrieve published knowledge articles
    • Compare the results and identify gaps
    • Present a numbered list of gaps sorted by incident volume
  • Expand and read through the AI agent's decision log to see each step:
    • Thought: The agent's reasoning about what to do next
    • Action: Which tool it decided to call
    • Action Inputs: The parameters it passed to the tool
    • Watch how the agent decides which keywords to search for in the KB based on what it found in the incidents
    • See how it synthesizes the two data sources into a gap analysis
What to expect The agent will identify 3-7 topic areas where incidents exist but knowledge articles do not. The exact topics will depend on your instance's demo data, but common gaps include network connectivity issues, VPN configuration, email/Outlook problems, and hardware-specific troubleshooting.
Tip If the agent gives a generic answer instead of instance-specific data, the issue is usually in the Instructions or the tool descriptions. Go back and verify both reference the tool names exactly as configured.
Dive Deeper / Challenge

How could this agent be expanded beyond the lab? What if it could automatically create draft KB articles for each gap? What if it ran on a weekly schedule and emailed the Knowledge Manager a gap report?

Challenge: What other data sources would make this analysis more powerful? (Hint: incident resolution notes, customer satisfaction scores, or self-service search terms with zero results.)

    JavaScript 
    (function execute(inputs, outputs) {
    var groupSkillId;
    var action_skill_id;
    var recSysId = UXCGAFUtil.getSysId(inputs.table, 'number', inputs.record_number);

    // Map the skills based on table
    var tableMapping = {
        'incident': { group: UXCConstants.ITSM_GROUPING, action: UXCConstants.ITSM_ACTION_STRATEGY },
        'sn_customerservice_case': { group: UXCConstants.CSM_GROUPING, action: UXCConstants.CSM_ACTION_STRATEGY },
        'sn_hr_core_case': { group: UXCConstants.HR_GROUPING, action: UXCConstants.HR_ACTION_STRATEGY }
    };

    var config = tableMapping[inputs.table];
    if (config) {
        groupSkillId = config.group;
        action_skill_id = config.action;
    }

    var grpSysId = UXCGAFUtil.getGroupSysId(recSysId, inputs.table, inputs.record_number, groupSkillId);
    var recs = grpSysId ? UXCGAFUtil.getGAFSimilarRecs(grpSysId, groupSkillId) : UXCGAFUtil.getRecReferences(inputs.table, recSysId);

    outputs.references = JSON.stringify(recs);

    // Extract IDs - Handling potential casing issue (sys_id vs sys_Id)
    var sysIds = [];
    if (Array.isArray(recs)) {
        recs.forEach(function(item) {
            var id = item.sys_id || item.sys_Id;
            if (id) sysIds.push(id.toString());
        });
    }

    outputs.sys_ids = JSON.stringify(sysIds);

    if (sysIds.length === 0) {
        outputs.outage_details = "No related records found.";
        return;
    }

    // --- Optimized Outage Retrieval ---
    var reportLines = [];
    var outageGR = new GlideRecord('cmdb_ci_outage');
    outageGR.addQuery('task_number', 'IN', sysIds); // Batch query is much faster
    outageGR.orderBy('task_number');
    outageGR.query();

    var currentTask = "";
    while (outageGR.next()) {
        // Grouping header in the report
        if (currentTask != outageGR.task_number.getDisplayValue()) {
            currentTask = outageGR.task_number.getDisplayValue();
            reportLines.push("\n--- Outages for Task: " + currentTask + " ---");
        }

        reportLines.push("  ----------------------------------------");
        reportLines.push("  Outage: " + outageGR.getDisplayValue());
        reportLines.push("  Begin: " + outageGR.begin.getDisplayValue());
        reportLines.push("  End: " + outageGR.end.getDisplayValue());
        reportLines.push("  Description: " + (outageGR.description || "No description"));
        reportLines.push("  ----------------------------------------");
    }

    outputs.outage_details = reportLines.length > 0 ? reportLines.join('\n') : "No outages found for these records.";

})(inputs, outputs);

Section 3.1: Incident Summarization #

This section guides you through ServiceNow's incident summarization feature using generative AI capabilities within the Service Operations Workspace.

  1. Return to Lab Instance
    Navigate back to your lab instance by clicking the ServiceNow logo in the upper-left corner, or remove any portal suffix from your instance URL.

  2. Access Service Operations Workspace

    • Select the Workspace tab
    • Choose Service Operations Workspace, which offers a consolidated view for managing task records including incidents, requests, and walk-ups

  3. Locate Incidents

    • Select list view
    • Under incidents, select All to display the complete incident list

  4. Activate AI Filter
    Click the Filter button featuring the Gen AI Sparkle icon in the upper-left area of the list view.

  5. Create Natural Language Filter

    • Copy this search criteria: "Any incident that has a description that contains Versioning errors"
    • Watch as Now Assist automatically constructs the filter
    • Select Apply to execute the query

  6. Open Incident
    Click the incident number link to open it within the Service Operations workspace. (Note: Your incident number may differ from examples shown)

  7. Generate Summary
    Select the Summarize button to leverage generative AI for incident analysis.

    The summarization feature examines short descriptions, full descriptions, work notes, and related records to produce summaries covering the issue, SLAs, impacted services, and completed actions. This proves valuable when dense documentation would otherwise require extensive reading time.

  8. Review Summary Actions
    The generated summary includes interactive elements:

    • Thumbs up/down icons for feedback and model training (if data sharing is enabled)
    • Copy functionality for clipboard access
    • Regenerate option for alternative summaries

  9. Share Summary
    Select the Share button to prepare the summary for integration into work notes.

  10. Edit and Save

    • Modify the summary by adding bulleted items as needed
    • Select Save to work notes to persist your edits
    Note Edited responses contribute to model refinement when data sharing is enabled.

Appendix A2: Agent Ideas #

Access Request Triage Agent

AspectDetails
NameAccess Request Triage Agent
DescriptionThe purpose of this agent is to evaluate employee IT access requests and produce a structured triage summary to support consistent, risk-aware approval decisions.
AI Agent RoleThe agent operates as a thorough but concise IT Security Analyst — professional and clear in communication, ensuring both IT staff and business managers can understand the output without technical jargon.
InstructionsThe agent follows a five-step process: (1) if access request details have not been provided, ask the user for the system or application being requested, the requestor's name, and their job role or department; (2) generate a plain-language summary of what is being requested and its likely business purpose; (3) assign a risk classification of Low, Medium, or High based on system sensitivity and role alignment; (4) identify any potential separation of duties conflicts based on the requestor's role; (5) recommend an approval path of Auto-Approve, Manager Approval Required, or Security Review Required, and if Medium or High risk, provide up to three clarifying questions the approver should ask before deciding.

IT Change Readiness Briefing Agent

AspectDetails
NameIT Change Readiness Briefing Agent
DescriptionThe purpose of this agent is to assess a submitted change request and generate a structured CAB readiness briefing that helps the Change Advisory Board make a fast, informed go/no-go decision.
AI Agent RoleThe agent operates as a precise and organized Change Analyst — structured and direct, producing output that is scannable and decision-ready for CAB members within two minutes of review.
InstructionsThe agent follows a five-step process: (1) if change request details have not been provided, ask the user to describe the change, the systems or configuration items affected, the planned maintenance window, and whether a rollback plan exists; (2) generate a one-paragraph executive summary of the change written for a non-technical CAB audience; (3) assign a risk rating of Standard, Significant, or Emergency based on scope and blast radius; (4) produce a readiness checklist confirming whether a rollback plan, maintenance window, assigned implementer, and stakeholder notifications are all in place; (5) provide a recommended CAB decision of Approve, Approve with Conditions, or Return for Revision with a one-sentence rationale and up to three risk factors the CAB should probe during review.

IT Vendor Escalation Briefing Agent

AspectDetails
NameIT Vendor Escalation Briefing Agent
DescriptionThe purpose of this agent is to help IT teams rapidly assemble a complete, professional escalation brief to submit to a third-party technology vendor when an issue cannot be resolved internally.
AI Agent RoleThe agent operates as a structured and detail-oriented IT Escalation Specialist — efficient and precise, producing only what the vendor needs to diagnose and prioritize the issue without unnecessary narrative.
InstructionsThe agent follows a six-step process: (1) if escalation details have not been provided, ask the user to describe the issue, the vendor and product involved, how long the issue has been occurring, and what troubleshooting steps have already been attempted; (2) generate a concise issue title suitable for a vendor support ticket subject line; (3) produce a structured problem statement covering what is broken, when it started, and the business impact; (4) compile a chronological list of troubleshooting steps already completed and their outcomes; (5) assign a severity classification of Sev1 through Sev4 with a one-sentence justification; (6) provide up to three questions or data requests the IT team should anticipate from the vendor so they can prepare responses before the support call begins.
